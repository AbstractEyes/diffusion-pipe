# Qwen Synthetic Characters

A dataset of **60,847 fully synthetic (AI-generated) human portrait/character images** produced
with **Qwen-Image** + the **Qwen-Image-Lightning** 4-step LoRA, with a prompt-augmentation policy
designed to give **balanced demographics, diverse facial expressions, and varied attributes** — and
to counter the base model's tendency to default to a narrow set of faces.

> [!IMPORTANT]
> **These are not real people.** Every image is generated by a diffusion model from a text prompt;
> any resemblance to a real individual is coincidental. The demographic columns (`race`, `gender`,
> `age_band`, `hair`, `eye`, …) record the **generation *intent*** — the attributes written into the
> prompt — **not measured/verified attributes of the rendered image.** The model does not always
> faithfully render the requested attribute. Do not treat these columns as ground-truth annotations.

## What's in it

- **60,847 images**, 1024-region resolutions: `1024×1024` (34.3%), `832×1216` (33.3%), `1216×832` (32.4%).
- Two provenance groups (see `id` / `source_prompt`):
  - **40,847** generated from the LLaVA face-description prompts in
    [`AbstractPhil/ffhq_with_llava_shorter_captions_flux_latents`](https://huggingface.co/datasets/AbstractPhil/ffhq_with_llava_shorter_captions_flux_latents)
    (numeric `id`; **text prompts only — no FFHQ images are used or redistributed**).
  - **20,000** generated from **synthetically composed** character captions (`id` prefix `synth_b…`):
    combinatorial composition × setting × clothing × activity × lighting, for contexts the face-only
    source lacks.

The `image` column is a standard 🤗 `datasets` **Image** feature (PNG), so the Hub viewer renders
thumbnails and `load_dataset` decodes it directly.

Every row also carries **three image-grounded captions of the generated portrait** — `caption_joycaption`
(detailed natural language, JoyCaption), `caption_qwen35_4b` (natural language, Qwen3.5-4B), and
`caption_animetimm` (booru-style tags, anime/wd-tagger) — alongside the `prompt` used to generate it
and the original `source_prompt`. Unlike `prompt`/`source_prompt` (generation *intent*), these three
describe the **rendered** image.

A lightweight, instantly-viewable 100-row preview of each rank lives under `dataset_viewer/` and is
exposed as the `rank0` / `rank1` viewer configs (the full `default` config renders in the paginated
viewer but has no inline first-rows preview because of its size).

## How it was generated

- **Models:** [`Qwen/Qwen-Image`](https://huggingface.co/Qwen/Qwen-Image) (~20B, Apache-2.0) +
  [`lightx2v/Qwen-Image-Lightning`](https://huggingface.co/lightx2v/Qwen-Image-Lightning)
  `Qwen-Image-Lightning-4steps-V1.0.safetensors`.
- **Inference:** 4 steps, `true_cfg_scale=1.0`, bf16, resident on 96 GB GPUs.
- **Prompt policy (`policy_version = augment-v2`)** rewrites each base caption:
  - **Race/ethnicity balancing** — when a prompt does not state one, a label is sampled from a
    tunable, fairly-uniform distribution (to counter the model's strong default toward a single
    appearance); explicit source races are respected.
  - **Expression diversity** — the source expression is stripped and a diverse expression sampled
    (the source captions say "smiling" ~67% of the time; here the smile-family is ~16% across 30
    distinct expressions).
  - **Attribute injection** — hair color, eye color (lightly conditioned on the sampled race),
    makeup (women), and jewelry-or-none are injected when unspecified; explicit ones are kept.
  - **Quality tier** — ~90% realistic color photography, ~10% deliberately "amateur" (slightly soft /
    grainy) for structural variety (`is_amateur`).
  - **Color enforcement** — every prompt carries a color cue and black-and-white terms are stripped,
    so the set contains **no monochrome/greyscale** images.
  - **Near-duplicate suppression** — a MinHash/LSH token-shingle similarity guard resamples
    attributes when a prompt is too similar to an earlier one.
  - Generation is **deterministic per `id`** (the `seed` column), so any row is reproducible.

## Dataset structure

| column | type | meaning |
|---|---|---|
| `id` | string | stable key; numeric = FFHQ-prompt-derived, `synth_b{N}_…` = synthetic-caption |
| `image` | Image | the generated PNG |
| `image_width`, `image_height` | int32 | pixel dimensions |
| `prompt` | string | the **augmented** prompt actually used for generation |
| `source_prompt` | string | the original base caption before augmentation |
| `race` | string | **intended** race/ethnicity label (generation intent, not verified) |
| `race_injected` | bool | whether the race was sampled (true) or taken from the source (false) |
| `is_tail` | bool | race is from the deliberately-rare "long tail" set |
| `gender` | string | intended subject: `woman` / `man` / `person` |
| `age_band` | string | always `25-35` (prompt-level constraint; see caveat below) |
| `hair`, `eye` | string | intended hair / eye attribute |
| `expression` | string | intended facial expression |
| `makeup`, `jewelry` | string | intended makeup (women) / jewelry (empty = not specified) |
| `is_amateur` | bool | rendered with a lower-quality "amateur" style |
| `seed` | int64 | deterministic per-`id` seed |
| `width_ratio` | string | e.g. `1024x1024` |
| `policy_version` | string | augmentation policy version (`augment-v2`) |
| `caption_joycaption` | string | image-grounded caption of the **generated** portrait (JoyCaption, detailed natural language) |
| `caption_qwen35_4b` | string | image-grounded caption of the **generated** portrait (Qwen3.5-4B, natural language) |
| `caption_animetimm` | string | image-grounded booru-style tags of the **generated** portrait (anime / wd-tagger) |

## Composition (actual, measured over all 60,847 rows)

- **Race (intended):** caucasian 28.1%, east_asian 15.8%, black 8.5%, south_asian 7.0%,
  hispanic/latino 6.1%, middle_eastern 5.2%, native_american 5.2%, southeast_asian 5.1%,
  multiracial 4.7%, pacific_islander 4.6%; long tail ~10% (central_asian, persian, polynesian,
  mestizo, scandinavian, mediterranean, ethiopian). 64% of labels were injected, 36% from the source.
- **Gender (intended):** woman 49.0%, man 43.7%, person 7.2%.
- **Expression:** 30 distinct; smile-family ~16.2% (top: neutral 6.1%, serious 5.3%, thoughtful 4.8%,
  soft smile 4.8%, pensive 4.1%, calm/confident 4.0% …).
- **Other:** 32 hair variants, 10 eye variants; makeup specified on ~48% of rows; "no jewelry" ~38%;
  amateur-quality ~10%.

> The set still skews **caucasian (28%) and east_asian (16%)** because the FFHQ-derived source
> captions frequently state those explicitly (which are respected). The 20k synthetic-caption rows,
> which have no source race, are distributed far more evenly.

## Intended uses

Research and development of generative image models: pretraining/fine-tuning data, data augmentation,
and studying demographic balance in synthetic face data.

## ⚠️ Limitations, biases, and required pre-processing

- **Age / minors — filter before use.** Prompts are constrained to adults (25-35) and minor terms are
  removed, but **no automated age verification has been applied** and a diffusion model can still
  render younger-looking faces. **Run a strong age-verification pass and remove any minor-appearing
  samples (and child-related tags) before using this dataset downstream.** This is a known, deliberate
  pending step — the dataset is *not* age-filtered as published.
- **Labels ≠ ground truth.** The demographic/attribute columns are prompt intent, not verified output
  attributes; do not use them to train or evaluate demographic classifiers or facial-recognition
  systems for real people.
- **Residual demographic skew** as noted above (caucasian/east_asian heavy on the FFHQ-derived half).
- **Synthetic artifacts.** Expect occasional diffusion artifacts (hands, accessories, text). ~10% of
  images are intentionally lower-quality ("amateur").
- **Not for identifying or impersonating real people.**
